# Authors CRUD Operations - Firebase Integration

This document describes the Authors CRUD (Create, Read, Update, Delete) operations implemented with Firebase for the LAAPS Institute project.

## Overview

The Authors management system allows admins to:

- Create new authors
- View all authors
- Search authors by name or email
- Update author information
- Delete authors
- Validate unique email addresses

## File Structure

```
lib/firebase/
├── authors.ts              # Authors CRUD operations
└── index.ts                # Export authors functions

app/api/
├── authors/
│   ├── route.ts            # GET all authors, POST new author
│   └── [id]/route.ts       # GET, PUT, DELETE individual author

app/admin/
└── authors/
    └── page.tsx            # Authors management UI
```

## API Endpoints

### GET `/api/authors`

Fetch all authors or search by name/email.

**Query Parameters:**

- `search` (optional): Search term for filtering authors

**Response:**

```json
{
  "success": true,
  "data": [
    {
      "id": "author-id",
      "name": "Dr. John Doe",
      "email": "john.doe@example.com",
      "bio": "Expert in research methodology",
      "expertise": ["Research", "Data Analysis"],
      "profileImage": "https://example.com/image.jpg",
      "publications": 25,
      "createdAt": "2026-03-11T10:00:00Z",
      "updatedAt": "2026-03-11T10:00:00Z"
    }
  ],
  "count": 1
}
```

### POST `/api/authors`

Create a new author.

**Request Body:**

```json
{
  "name": "Dr. Jane Smith",
  "email": "jane.smith@example.com",
  "bio": "Policy analysis expert",
  "expertise": ["Policy", "Governance"],
  "profileImage": "https://example.com/jane.jpg",
  "publications": 15
}
```

**Response:**

```json
{
  "success": true,
  "message": "Author created successfully",
  "data": {
    "id": "new-author-id",
    "name": "Dr. Jane Smith",
    "email": "jane.smith@example.com",
    ...
  }
}
```

**Validation:**

- Name and email are required
- Email must be unique across all authors

### GET `/api/authors/[id]`

Fetch a single author by ID.

**Response:**

```json
{
  "success": true,
  "data": {
    "id": "author-id",
    "name": "Dr. John Doe",
    ...
  }
}
```

### PUT `/api/authors/[id]`

Update an existing author.

**Request Body:** (all fields optional)

```json
{
  "name": "Dr. John Doe Updated",
  "email": "john.updated@example.com",
  "bio": "Updated bio",
  "expertise": ["Research", "Statistics", "ML"],
  "profileImage": "https://example.com/updated.jpg",
  "publications": 30
}
```

**Response:**

```json
{
  "success": true,
  "message": "Author updated successfully",
  "data": { ... }
}
```

**Validation:**

- If email is changed, it must be unique (excluding current author)

### DELETE `/api/authors/[id]`

Delete an author.

**Response:**

```json
{
  "success": true,
  "message": "Author deleted successfully"
}
```

## Data Model

### Author Interface

```typescript
interface Author {
  id?: string; // Auto-generated by Firestore
  name: string; // Required
  email: string; // Required, unique
  bio?: string; // Optional
  expertise?: string[]; // Optional array of expertise areas
  profileImage?: string; // Optional URL to profile image
  publications?: number; // Optional count of publications
  createdAt?: string; // Auto-generated timestamp
  updatedAt?: string; // Auto-generated timestamp
}
```

## Firebase Functions

Located in [`lib/firebase/authors.ts`](lib/firebase/authors.ts):

### `createAuthor(authorData)`

Creates a new author in Firestore.

**Parameters:**

- `authorData`: Author data (without id, createdAt, updatedAt)

**Returns:** Author document ID

**Example:**

```typescript
import { createAuthor } from "@/lib/firebase";

const authorId = await createAuthor({
  name: "Dr. Jane Smith",
  email: "jane@example.com",
  bio: "Expert researcher",
  expertise: ["Research", "Policy"],
  publications: 15,
});
```

### `getAuthor(authorId)`

Retrieves a single author by ID.

**Parameters:**

- `authorId`: The author's document ID

**Returns:** Author object or null if not found

### `getAllAuthors()`

Retrieves all authors, ordered by name.

**Returns:** Array of Author objects

### `getAuthorsByExpertise(expertise)`

Retrieves authors filtered by expertise area.

**Parameters:**

- `expertise`: Expertise area to filter by

**Returns:** Array of Author objects

### `searchAuthors(searchTerm)`

Searches authors by name or email (case-insensitive).

**Parameters:**

- `searchTerm`: Search string

**Returns:** Array of matching Author objects

### `updateAuthor(authorId, authorData)`

Updates an existing author.

**Parameters:**

- `authorId`: The author's document ID
- `authorData`: Partial author data to update

**Returns:** void

### `deleteAuthor(authorId)`

Deletes an author from Firestore.

**Parameters:**

- `authorId`: The author's document ID

**Returns:** void

### `checkAuthorEmailExists(email, excludeId?)`

Checks if an email is already in use by another author.

**Parameters:**

- `email`: Email to check
- `excludeId`: Optional author ID to exclude from check (for updates)

**Returns:** boolean

## Usage Example

### Frontend: Using the Authors API

```typescript
// Fetch all authors
const response = await fetch("/api/authors");
const { data: authors } = await response.json();

// Search authors
const searchResponse = await fetch("/api/authors?search=john");
const { data: results } = await searchResponse.json();

// Create author
const createResponse = await fetch("/api/authors", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    name: "Dr. Sarah Johnson",
    email: "sarah@example.com",
    expertise: ["Public Health", "Epidemiology"],
    publications: 18,
  }),
});

// Update author
const updateResponse = await fetch("/api/authors/author-id", {
  method: "PUT",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    publications: 19,
  }),
});

// Delete author
const deleteResponse = await fetch("/api/authors/author-id", {
  method: "DELETE",
});
```

### Backend: Direct Firebase Usage

```typescript
import {
  createAuthor,
  getAllAuthors,
  updateAuthor,
  deleteAuthor,
  searchAuthors,
} from "@/lib/firebase";

// In your server-side code or API routes
const authorId = await createAuthor({
  name: "Dr. Michael Brown",
  email: "michael@example.com",
  expertise: ["Economics", "Development"],
});

const authors = await getAllAuthors();
const results = await searchAuthors("smith");
await updateAuthor(authorId, { publications: 20 });
await deleteAuthor(authorId);
```

## Security Considerations

1. **Authentication**: Ensure only authenticated admin users can access these endpoints
2. **Authorization**: Implement role-based access control
3. **Validation**: Always validate input data on the server side
4. **Email Uniqueness**: Enforced at the application level
5. **Firestore Rules**: Configure appropriate security rules

### Recommended Firestore Rules

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /authors/{authorId} {
      // Allow read for authenticated users
      allow read: if request.auth != null;

      // Only allow write for admin users
      allow write: if request.auth != null &&
        get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin';
    }
  }
}
```

## Error Handling

All API endpoints return consistent error responses:

```json
{
  "success": false,
  "error": "Error message description"
}
```

Common error codes:

- `400`: Bad request (missing required fields)
- `404`: Author not found
- `409`: Conflict (email already exists)
- `500`: Internal server error
